# AEP Showcase Notes

Autonomous Arcade is designed to make autonomous engineering tangible.

The product should not merely describe AEP. It should be built by AEP, changed by AEP, and improved in ways that are visible to visitors.

## Public Framing

Recommended public framing:

> Autonomous Arcade is developed by the Autonomous Engineering Platform by CognicellAI.

Keep the wording short and confident. The arcade should feel like a real product first, with AEP presented as the delivery system behind it.

## What To Show

Good public signals include:

- recent shipped changes
- merged pull requests
- passing CI checks
- new games or mechanics
- small changelog entries
- links back to the repository

Avoid exposing internal operational details that would distract from the product or require authentication, secrets, moderation, or production support before the MVP needs them.

## Suggested Build Log Shape

The first build log can be static or generated from repository metadata.

Each entry can include:

- title
- short summary
- shipped date
- related pull request
- affected game or feature

Later, AEP can publish richer public events from its canonical project state.
